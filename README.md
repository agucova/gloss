# gloss

Browser extension + web app for highlighting text on web pages and sharing those highlights with friends. A spiritual successor to Curius.

## Stack

- **Backend**: [Convex](https://convex.dev) — schema, queries/mutations, HTTP actions, search indexes
- **Auth**: [Better-Auth](https://www.better-auth.com) via `@convex-dev/better-auth` (email/password, passkeys, optional Google/Apple/Resend magic-link)
- **Web**: React 19 + Vite + TanStack Router + Tailwind 4, deployed to Cloudflare Workers
- **Extension**: [WXT](https://wxt.dev) (Chrome + Firefox), Solid.js for content-script UI
- **CLI**: `packages/cli` (publishes `gloss` + `gloss-mcp` binaries, OAuth + PKCE flow against Convex HTTP actions)
- **Tooling**: Bun, Turborepo, oxlint + oxfmt, Lefthook, Vitest + `convex-test` (unit), Playwright (e2e)

## Monorepo layout

```
apps/
  web/          # React SPA → Cloudflare Workers
  extension/    # WXT browser extension (Chrome / Firefox)
convex/         # Convex functions, schema, HTTP routes, seed, unit tests
packages/
  anchoring/    # W3C Web Annotation selectors (range/text-position/text-quote)
  cli/          # @gloss/cli + MCP server (unpublished, untested end-to-end)
  config/       # Shared TypeScript configs
  curius/       # Reverse-engineered Curius client (highlight import)
  dashboard/    # Shared dashboard UI pieces
  fonts/        # Inter + font helpers
e2e/            # Playwright specs (web + extension projects)
```

## First-time setup

```bash
bun install

# Link or create a Convex dev deployment (prompts; writes .env.local).
# This also starts a watcher that pushes function/schema changes.
bunx convex dev
```

Set required env vars on the Convex backend (via dashboard or CLI — these are _not_ in `.env`):

```bash
bunx convex env set BETTER_AUTH_SECRET "$(openssl rand -base64 32)"
bunx convex env set SITE_URL http://localhost:3001
# Optional — enable if you want social login / magic links:
bunx convex env set GOOGLE_CLIENT_ID …
bunx convex env set GOOGLE_CLIENT_SECRET …
bunx convex env set APPLE_CLIENT_ID …
bunx convex env set APPLE_CLIENT_SECRET …
bunx convex env set RESEND_API_KEY …
bunx convex env set EMAIL_FROM "Gloss <noreply@yourdomain>"
```

Then seed (optional):

```bash
bun run convex:seed
```

## Develop

```bash
bun run dev          # convex dev + web + extension in parallel
bun run dev:web      # convex dev + web
bun run dev:ext      # extension only (hot reload on :5555)
```

Ports: web `:3001`, extension dev server `:5555`. Convex deployment URL comes from `.env.local` (`CONVEX_URL`, `CONVEX_SITE_URL`, `CONVEX_DEPLOYMENT`).

## Environment

All dev env vars live in `.env` (checked-in shape in `.env.example`). `.env.local` is auto-generated by `convex dev` and should not be committed.

Client-side (web + extension — all must be prefixed `VITE_`):

| Var                    | Notes                                                                      |
| ---------------------- | -------------------------------------------------------------------------- |
| `VITE_CONVEX_URL`      | `https://<slug>.convex.cloud` — for the React client                       |
| `VITE_CONVEX_SITE_URL` | `https://<slug>.convex.site` — Better-Auth base URL                        |
| `VITE_WEB_URL`         | e.g. `http://localhost:3001`, used by the extension for the login redirect |

Server-side (set on the Convex backend via `bunx convex env set`, not in `.env`):

| Var                             | Required | Notes                                  |
| ------------------------------- | -------- | -------------------------------------- |
| `BETTER_AUTH_SECRET`            | yes      | ≥32 chars, `openssl rand -base64 32`   |
| `SITE_URL`                      | yes      | CORS origin for the Better-Auth routes |
| `GOOGLE_CLIENT_ID` / `_SECRET`  | no       | Google OAuth                           |
| `APPLE_CLIENT_ID` / `_SECRET`   | no       | Apple OAuth                            |
| `RESEND_API_KEY` / `EMAIL_FROM` | no       | Magic-link emails                      |

## Scripts

| Command                                     | What it does                                             |
| ------------------------------------------- | -------------------------------------------------------- |
| `bun run dev`                               | Convex watcher + web + extension                         |
| `bun run build`                             | Turborepo build across workspaces                        |
| `bun run build:ext` / `build:ext:firefox`   | Extension builds                                         |
| `bun run zip:ext` / `zip:ext:firefox`       | Package extension for store upload                       |
| `bun run check-types`                       | `tsc --noEmit` across workspaces                         |
| `bun run lint` / `lint:fix`                 | oxlint (+ oxfmt on fix)                                  |
| `bun run format` / `format:check`           | oxfmt                                                    |
| `bun run test`                              | `convex-test` unit tests via Vitest (`convex/*.test.ts`) |
| `bun run e2e` / `e2e:web` / `e2e:extension` | Playwright                                               |
| `bun run convex:dev` / `convex:deploy`      | Convex dev watcher / prod deploy                         |
| `bun run convex:seed`                       | Run `seed:run` internal mutation                         |
| `bun run deploy:web` / `deploy:web:preview` | Wrangler deploy to Cloudflare                            |

Pre-commit hook (Lefthook) runs `oxfmt` + `oxlint` on staged files.

## Deploy

**Convex (backend)** — `bunx convex deploy` pushes to the prod deployment linked via `CONVEX_DEPLOY_KEY`. Remember to set the same env vars listed above on the prod deployment.

**Web (Cloudflare Workers)** — `bun run deploy:web`. `apps/web/wrangler.toml` routes `gloss.space/*` to the worker. Static assets served from `apps/web/dist` with SPA fallback. Build-time env needs `VITE_CONVEX_URL`, `VITE_CONVEX_SITE_URL`, `VITE_WEB_URL`.

**Extension** — `bun run zip:ext` (Chrome) / `bun run zip:ext:firefox`, then upload the ZIP to each store.

## Testing

- **Unit (Convex functions)**: `bun run test` — uses `convex-test` with Vitest's edge-runtime environment. Test files live alongside the Convex functions (`convex/*.test.ts`). Each test passes `import.meta.glob("./**/!(*.test).*s")` as the second arg to `convexTest` so it resolves `_generated/` under Bun's hoisted `node_modules`.
- **E2E**: `bun run e2e` — Playwright with two projects (`web`, `extension`). Fixtures in `e2e/fixtures/`, specs under `e2e/web/` and `e2e/extension/`.
